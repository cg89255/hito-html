JSON


Historia

JSON fue creado por Douglas Crockford a principios de la década de 2000 como un formato de intercambio de datos ligero y fácil de leer. Se basó en una parte de la sintaxis de JavaScript, pero se puede utilizar con cualquier lenguaje de programación. JSON surgió como una alternativa más simple y fácil de usar en comparación con XML para el intercambio de datos estructurados en la web y en aplicaciones


Normas de uso

  • Sintaxis simplificada: JSON utiliza una sintaxis más simple que XML, basada en pares de clave-valor.
  • Ligero: JSON es más ligero que XML, lo que significa que los datos en formato JSON tienden a ser más compactos.
  • Fácil de leer y escribir: Debido a su sintaxis simple, JSON es fácil de leer y escribir tanto para humanos como para máquinas
  • Independiente del lenguaje: JSON puede ser utilizado con una amplia variedad de lenguajes de programación y plataformas
  • Soporte integrado en JavaScript: JSON es nativo en JavaScript, lo que significa que es fácil de trabajar con él en aplicaciones web basadas en JavaScript

Sintáxis

La sintaxis básica de JSON consiste en pares de clave-valor separados por comas, con objetos encerrados entre llaves {} y arrays encerrados entre corchetes [].


Aplicación empresarial

  • API RESTful: JSON se utiliza como formato de intercambio de datos en las API RESTful para la comunicación entre clientes y servidores
  • Almacenamiento de datos: JSON se utiliza para almacenar datos estructurados en bases de datos NoSQL y documentos de bases de datos como MongoDB
  • Comunicación entre microservicios: JSON se utiliza para intercambiar datos entre microservicios en arquitecturas de microservicios debido a su ligereza y facilidad de uso
  • Integración de sistemas: JSON se utiliza para el intercambio de datos entre sistemas heterogéneos en entornos empresariales, especialmente cuando se requiere una integración ligera y ágil
